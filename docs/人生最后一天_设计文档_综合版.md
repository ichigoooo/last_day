# 人生最后一天 设计文档（综合版）

- 产品名称：人生最后一天  
- 英文名称：Last Day Simulator  
- 文档版本：综合版（PRD V1.0.1 + 玩法设计 V2.0 + **最后一天自由行动四机制**）  
- 文档定位：产品愿景、体验原则与**可落地的玩法机制**合一的设计稿  

---

## 0. 文档说明

本版在《人生最后一天》原设计文档（叙事与仪式框架）基础上，融入《玩法设计 V2》的**规则—资源—约束**层，解决「仅靠阅读与选项难以形成真实紧迫感」的问题。

**综合原则：**

- **气质与目标**仍以原设计为准：黑色幽默、仪式感、清醒但不伤人、可分享、轻度疗愈。  
- **结构与节奏**以 V2 的阶段划分为准：入殓登记 → 宣判 → **最后一天（核心玩法）** → 死亡 → 葬礼 → 冥想 → **终局（单次体验收束）**。  
- **「最后一天」的体验形态**：用户**自由行动**——可在游戏内去任何场所、做任何事；由**时间持续流逝、现金余额、手机电量、场景内自然语言行动**四条机制共同约束。死神提供**绿色通道传送**降低「跑图」成本，但无法买回时间本身。叙事**嵌入场所与行动**，由 AI 旁白与选项推进，而非单独一条纯文本线。  

---

## 1. 产品概述

《人生最后一天》是一款带有黑色幽默、互动叙事和自我反思属性的体验型产品。它用「最后一天」这个极端设定，帮助用户把注意力从日常琐碎和情绪内耗中抽离出来，重新看见自己真正重视的东西。

产品的核心不是制造恐惧，而是借由「有限时间」这个设定，推动用户完成一次有仪式感的自我审视。用户会经历**入殓登记与死因宣判**、在**可量化的游戏内「最后一天」**里自由取舍（时间、金钱、电量与行动）、**虚拟葬礼与死亡冥想**、**终局收束（可选一句承诺或留白）**，借由机制结果与叙事回声形成既荒诞又清醒的完整体验。本产品定位为**单次完整体验**，不强调多局累积或「轮回」叙事。

## 2. 产品目标

### 2.1 核心价值

- **自我观照**：通过时间/现金/电量约束下的真实取舍、场景内「想做什么」的自然语言表达、悼词与回声对原话的引用，帮助用户回看选择背后的动机与代价。  
- **生命教育**：用「最后一天」的情境，逼近什么才是真正重要的。  
- **认知松动**：借死神批注、地点叙事与仪式收束，用清醒但不羞辱人的方式，松动自我欺骗、逃避与「改天再说」的惯性。  
- **轻度疗愈**：通过叙事、冥想和书写，提供温和的情绪整理空间。  
- **可分享性**：让葬礼通知、轨迹、讣告卡、体验简报等内容具备传播属性，形成用户自发分享。  

### 2.2 产品气质

- 语气：冷静、克制、带一点荒诞感。  
- 风格：黑色幽默、仪式感、轻叙事、强代入。  
- 体验目标：让用户「被戳一下」，但不是被伤害。  

### 2.3 体验原则（延续原设计）

- 有选择感，但不允许用户逃离主题。  
- 有戏剧性，但不走纯搞笑路线。  
- 有反思，但不把用户变成被审判对象。  
- 有分享欲，但不靠社交压力驱动。

## 3. 目标用户与场景

### 3.1 目标用户

- 内耗打工人：长期处于高压、疲劳、自我怀疑中，需要安全空间整理情绪。  
- 叙事与策略向玩家：喜欢互动小说，也愿意在**资源约束**下做取舍。  
- 自我探索者：对心理学、哲学、决策科学感兴趣。  
- 情绪节点用户：分手、失业、失败等节点上，需要能承住情绪、又不必在社交里强撑体面的一场独处仪式。  

### 3.2 使用场景

- 深夜独处：失眠、焦虑、不想工作时。  
- 重大决策前：换工作、分手、创业方向选择时。  
- 情绪低谷期：被拒绝、被批评、遭遇失败时。  
- 社交分享后：把结果发给朋友，形成话题和讨论。  

### 3.3 用户痛点（原设计）

- 缺少能逼自己把「真正在意的事」说清楚、又不必对他人交代的契机。  
- 朋友通常只会安慰，或不敢说真话。  
- 自己容易陷入思维循环。  
- 想反思，但不知道从哪里开始。  
- 缺少能标记人生转折的仪式感。  

**玩法层补充：** 许多产品只让用户「读到一个关于时间不够的故事」；本综合版要求用户**在系统里真的感到时间不够、钱不够花完或花错、电量不够、事做不完**，使「后悔」来自机制而不仅是文案。

## 4. 体验架构：从叙事驱动到机制驱动

| 维度 | 原设计侧重（V1） | 综合版（融入 V2） |
|------|------------------|-------------------|
| 核心循环 | 文本推进 → 2～3 选项 → 回收 | **自由选点 → 绿色通道传送（耗时）→ 场景内自然语言行动 → AI 旁白与选项/自定义** |
| 紧迫感来源 | 叙事与氛围描写 | **游戏内时间恒逝、行动耗时、现金与电量双约束** |
| 选择的意义 | 影响语气与片段，分支不无限裂变 | **分配时间、5000 元现金、电量与「做什么」：去哪、联系谁、钱花哪** |
| 「失败」 | 弱化为情绪体验 | **可发生**：来不及做完、钱剩或花错、没电、消息未及时收到回复 |
| 叙事位置 | 承上启下的核心段落 | **嵌入各「场所」内**，由用户意图（自然语言）+ 遗愿骨架共同驱动 |
| AI 职责 | 生成贯穿全线的文本 | **预制骨架 + 关键处个性化插入**（见第 11 章） |

**一句话：** 原设计保证「说什么、什么气质」；机制层保证「**为什么玩家会难受、会后悔**」来自**时间恒逝、现金与电量有限、行动不可兼得**。叙事与仪式负责消化困境。

## 5. 完整体验流程（综合）

### 5.1 阶段总览

| 阶段 | 建议真实时长 | 内容概要 | 情绪目标 |
|------|----------------|----------|----------|
| 入殓登记 | 2～3 分钟 | 表格化三问 + 死神短批注 | 好奇 → 微微不安 |
| 宣判 | 1～2 分钟 | 死因通知（隐喻关联）+ **一件事遗愿**（优先自由输入） | 荒诞 → 接受设定 |
| **最后一天** | **12～15 分钟** | **自由行动：时间恒逝 + 绿色通道 + 场景自然语言行动 + 5000 元现金 + 手机（电量/消息/朋友圈）** | **真实困境与取舍** |
| 死亡 | 约 30 秒～1 分钟 | 界面「死去」、纯黑静候 | 空 |
| 葬礼 | 2～3 分钟 | 悼词、墓志铭、遗言、讣告卡 | 庄重 → 被看见 |
| 冥想 | 3～5 分钟（可跳过） | 结构化引导 + 个性化回扣 | 沉淀 |
| 终局 | 1～2 分钟 | 界面复苏感 + **一句承诺（可选）**；返回入口 | 清醒 → 微酸的力量感 |

**整轮建议：约 18～25 分钟**。快速模式可通过缩短最后一天游戏内可玩时长、跳过冥想等方式压到约 10～15 分钟，具体由产品配置决定。  

### 5.2 流程节奏（原则）

- 情绪曲线：**进入世界观 → 机制化压迫 → 集中回收 → 安静落地**。  
- 情绪高点处保留停顿，勿一路狂推。  
- 支持中断与恢复：宜从「最近一个地点/移动段」或检查点继续。  

### 5.3 重复可玩性（综合）

- 死因：由 LLM 结合入殓登记生成；再次游玩为**新的一局**，与历史存档无绑定叙事。  
- 遗愿与路线：不同遗愿类型、不同传送顺序与到访场所、不同时间/现金/电量策略。  
- 手机：主动发信对象与朋友圈内容不同；回复延迟随机，每局体验不同。  
- **不做多局档案累积**：不提供「轮回簿」式长期回看；单次通关后可从主菜单重新开始，运行时数据重新初始化。

## 6. 最后一天：自由行动与四大机制（定稿）

**定位：** 「最后一天」阶段用户可**自由行动**：去任何场所、做任何事（在叙事与系统允许范围内）。以下四条机制同时生效，共同构成决策空间。

### 6.1 机制一：时间（恒逝）

- **规则：** 游戏内时间在流动；**无论用户正在传送、读屏、犹豫、看旁白还是操作手机，时间都在流逝。** 不存在「暂停在世界之外」的安全区（读档、菜单策略除外，由产品定义）。  
- **表现：** 顶部**时钟弧线**，日光自左向右推进；游戏内时间相对真实时间加速（例如约 **1 真实分钟 ≈ 1 游戏小时**，可调平衡）。  
- **归零：** 当日时间耗尽，未完成的行动即告中断，进入死亡阶段。  
- **日光阶段表**（统一 UI/叙事节奏）：清晨暖光 → 正午强烈 → 下午偏黄 → 黄昏橘红 → 深夜暗蓝 → 午夜至黑。紧迫感以光感 + 恒逝机制为主，而非单纯读秒。  

### 6.2 机制二：行动（绿色通道 + 场景内自然语言）

#### 6.2.1 自由目的地与绿色通道

- 用户可在地图/场所列表中**自由选择**要去的地方（场所可与灵魂画像关联预生成，见 7.1；亦允许用户表达「想去××」由系统映射）。  
- **死神绿色通道：** 用户选定目的地后，死神为其开辟**快速传送**，使用户抵达该场所；**每次传送固定消耗游戏时间 30 分钟**（不占距离差，强调「被允许瞬间到位仍要付时间」）。  
- 若产品保留「非绿色通道」的移动方式，其时间成本应**不低于**绿色通道或仅作叙事皮肤，避免架空 30 分钟规则；**以本文档绿色通道为默认主路径。**  

#### 6.2.2 场景内：想做什么 + AI 旁白 + 选项或自定义

- 到达场所后，用户用**自然语言**描述「想在这里做什么」（短句即可）。  
- **AI：** 根据用户输入与灵魂画像/遗愿，生成**少量旁白**（氛围与事件推进），并给出：  
  - **一段由系统随机出的时间消耗**（表示这件事在游戏内会占去多久，随机区间与场所/意图标签绑定，具体数值表由策划填）；  
  - **三个备选行动选项**（态度或策略不同，便于代入）。  
- **用户决策：** 从 **三个选项中选一个**，或**输入自己的文字**作为**自定义行动**（自定义同样消耗时间，时间可由 AI 结合意图估一个数，或落入预设随机区间，需与「随机时间」规则一致、可追溯）。  
- 时间开始流逝后，进入下一段或允许再次描述「接下来做什么」，循环直至用户选择传送至下一场所或时间耗尽。  

### 6.3 机制三：金钱（5000 元）

- **设定：** 用户「来不及取更多钱」，身上**固定现金 5000 元**（单位与文案随地区，默认人民币）。本局无额外取款渠道。  
- **花费：** 用户可与**死神对话**，在死神提供的**花钱选项**中选择（如：给某人买礼物、捐款、最后一顿饭、委托一件事等）；选中即扣款并触发对应叙事或系统结果。  
- **设计要点：** 花钱应带来**可见后果**（叙事片段、关系反馈、结算展示），避免沦为纯数值；余额可在 UI 常驻显示。  
- **边界：** 金额花完即无法再选需付费选项；剩余现金可在葬礼/总结中呈现「带走了多少没花出去」。  

### 6.4 机制四：电量与手机（100% 起）

- **初始电量：100%**（本局上限，不再用 15% 开局旧设定）。  
- **发消息：** 用户可向**任意对象**发送消息（输入称呼或选联系人，由产品与 AI 约束合理性）。**发出后**需等待对方「回复」：**在发起大模型生成回复请求时，随机采样一段延迟时间，表示游戏内等待时长，范围为 0～180 分钟（游戏时间）**——模拟「不知道何时回信」的不确定性。  
- **朋友圈：** 用户可**发朋友圈**（图文或纯文由产品定）；是否额外消耗时间或电量可细化，默认至少占用**当前场景时间流逝**（发圈过程时间仍走）。  
- **亮屏耗电：** **手机处于亮屏期间持续耗电**；**每经过 3 分钟游戏时间，扣除 1% 电量**（即亮屏时 **3 游戏分 / 1%**）。息屏策略（关屏是否暂停扣电）由产品统一：若关屏不扣电，则用户可通过关屏省电，但可能错过消息提示——形成取舍。  
- **充电宝充电：** 用户可选择**寻找并使用充电宝**为手机补电。**固定消耗 1 小时游戏时间**（计入当日总时长，时间照常流逝；可用短叙事表现「找桩/等待充电」）。充电完成后**恢复多少电量**由数值表配置（例如一次充满至 **100%**，或按固定百分比回复），且**不超过本局上限 100%**。是否允许重复充电、充电宝是否需花钱或仅耗时，由产品定；若与死神/场所联动，需在 UI 上标明「−1 小时」。  
- **没电后果：** 电量归零后可定义关机：无法再看屏、发消息等；未收到或未发出的内容进入结算展示。语气保持克制，不嘲弄用户。  

### 6.5 四大机制如何咬合

**核心决策层：** 在**时间恒逝**的前提下，如何分配 **5000 元**、**亮屏与发信/发圈**、是否**花 1 小时找充电宝补电**、**传送至哪些场所**、以及在每个场所**选系统选项还是自定义行动**。  

**循环示意（示例）：** 看时间与余额/电量 → 选择去某处（**−30 分钟**）→ 自然语言说想做什么 → AI 旁白 + 随机耗时 + 三选项或自定义 → 时间推进；并行可能：**发消息（等待 0～180 分钟内随机时刻的回复）**、**与死神花钱**、**发朋友圈（亮屏耗电）**、**找充电宝充电（−1 小时游戏时间，补电）** → 回到决策。

## 7. 各环节功能设计（融合两版）

### 7.1 入殓登记与灵魂画像（V2 前置）

**定位：** 不是闲聊，而是一张旧表格式的**轻输入**，产出 **Soul Profile（灵魂画像）**，贯穿手机消息、地图标签、叙事隐喻与悼词引用。

**建议三问：**

1. 你每天花最多时间做的事是什么？（对应「公司/学校」等权重）  
2. 若此刻有人给你发消息，你最希望是谁？（对应「那个人」与相关消息）  
3. 哪件事你对自己说「改天再说」已超过三次？（对应「故地」、逃避议题与回声引用）  

**交互：** 提交后死神在批注栏写 **1～2 句**点评（第一次「被看见」），须冷静、可隐喻，不嘲弄。  

### 7.2 死因预警与宣判

**与原设计一致处：**  
- 视觉如旧纸通知/印章感；气质荒诞、轻黑色幽默；避免血腥与恶意；**初次体验简洁**。  
- 死神开场白：承认荒诞、非惩罚、冷静旁白；可保留「人类总假装自己不会死」类句式。  

**与 V2 一致处：**  
- 死因从预制库（建议 **30～50 条**）中选取，并与登记表核心议题建立**隐喻关联**，用户未必自觉意识到。  
- **最近出现过的死因可暂时移出池子**，避免连续重复（沿用原设计防重复逻辑）。  

### 7.3 遗愿确认

**综合取舍：**  
- 死神明确：**一天只够做一件事层面的核心**——与 V2「不是一天很多事」一致。  
- **优先自由输入**；若卡住，再给出 **3 条**基于灵魂画像的个性化建议（取代原「大量预设遗愿列表」为唯一主路径，减少菜单感）。  
- 原设计中「换一换查看更多遗愿」可作为**可选补充**（非首屏主路径），以免与「一件事」焦点冲突。  

### 7.4 最后一天叙事（嵌入场所与自由行动）

- 叙事不单独成段，而在用户**传送抵达场所之后**，随 **6.2.2** 的流程展开：用户自然语言意图 → AI 旁白 → **随机时间成本** → **三选一或自定义行动**。  
- AI 仍可按**遗愿类型**选用底层叙事骨架（告别、完成、表白、逃离、什么都不做等），在骨架内填充旁白与选项文案。  
- **对话拼图**（见第 8 章增强 C）可与「三选项 / 自定义」并存：关键对话用碎片句，其余行动仍用三选项或自定义。  
- **配图**：仍须克制，服务当前场所情绪；可与日光阶段联动。  
- **时间耗尽**：当前行动可截断，用短句标示「没做完 / 来不及了」；若消息回复延迟大于剩余时间，则**收不到回信**——力量来自**机制结果**。  
- **死神花钱线**：与 6.3 联动，叙事上需解释「为何通过死神消费」（黑色幽默或契约感，避免纯菜单感）。  

### 7.5 日程总结、葬礼与分享

**总结层：**  
- 整合：遗愿、到访场所与关键行动、自然语言/自定义选择摘要、**现金结余与主要消费**、**手机电量结局**、消息发出/回复情况、一句总体判断。  
- **葬礼通知 / 讣告卡**（两版融合）：  
  - 字段可含：时间、死因、旅程摘要、对来宾的要求、用户补充的遗憾与感受。  
  - V2 补充：**悼词**（第三人称、≤200 字、至少一处用户原话引用）、**墓志铭三选一或自填**、**遗言**一句。  

**分享：**  
- 视觉焦点明确、一句可记住的总结；轨迹图、讣告卡均可为分享载体。  

### 7.6 死亡瞬间（阶段过渡）

- 时间归零 → 声音淡出 → **界面死去**（见第 8 章增强 A）→ 纯黑静候数秒 → 无操作或极简反馈文案。  
- 与原设计「仪式高潮前的收束」一致，但**感官层更明确**。  

### 7.7 死亡冥想

**结构建议融合 V2：** 降落 → 回放本局关键时刻（意象化）→ 短空白 → **回声**（引用用户核心输入）→ 归来现实。  

**语气遵循原设计：** 轻、慢、具体意象、结尾回到「明天还要继续活」。  
**可跳过**，跳过时不道德绑架。个性化内容由 AI 在骨架内**少量插入**。  

### 7.8 终局书写与承诺

- **原设计的半开放式引导句**仍适用，作为可选多条备忘。  
- **V2 的单一承诺句**可作为收束主交互：明天第一件要做的不一样的事是什么（一句即可，可不写）。  
- **单次体验**：不累积多局「人生档案」；再次游玩为新的一局。

## 8. 辅助增强机制（可选，按优先级）

| 优先级 | 名称 | 要点 |
|--------|------|------|
| 高 | **界面正在死去** | 随时间接近午夜，饱和度、音效、触控延迟、UI 消退；死亡与终局「亮起来」瞬间强对比。 |
| 中 | **记忆行囊** | 3～4 格，拾取记忆碎片；满则丢弃，丢弃有仪式；葬礼展示「带进死亡的」与「选择遗忘的」。 |
| 中 | **对话拼图** | 与「那个人」对话时从碎片中选 2～3 句说出，组合影响走向，无唯一正确答案。 |
| 低 | **亡者回声** | 匿名墓志铭/遗言/选择以弱形式出现在他人局中；需后端，可后续迭代。 |

## 9. 视觉与交互风格（综合）

### 9.1 整体（原设计）

- 主背景偏深，仪式感；辅助色克制金、冷灰蓝、低饱和红。  
- 卡片与层级清晰，避免花哨。  

### 9.2 模块气质

- 死因/宣判：旧纸通知。  
- 入殓登记：桌面表格、笔触与纸张声（可音效辅助）。  
- **地图 / 场所**：手绘示意、图标+标签；可选展示传送轨迹与到访记录。  
- **手机**：简约系统风；聊天、**朋友圈**、亮屏与电量条清晰可辨；**充电宝**入口与「将消耗 1 小时游戏时间」提示明确。  
- **死神消费**：与对话或契约式 UI 结合，避免纯商城列表。  
- 叙事区：文本为主，打字机感可选。  
- 葬礼：礼堂聚光；终局：晨光、清晰、与「死去」界面反差。  

### 9.3 交互原则（原设计 + 玩法）

- 每步有反馈；选择面保持克制。  
- **机制层**：玩家随时能理解「还剩多少时间、多少现金、多少电量」。  
- 支持中断与恢复。

## 10. 风险与边界 + 情绪安全（融合）

### 10.1 内容与人身边界（原设计）

- 叙事、批注与悼词不做人身攻击；不替代心理咨询；沉重主题需克制。  
- 提供温和模式或弱化表达；关键免责声明可见。  

### 10.2 体验风险（原设计）

- 语气过重易冒犯；反思过浅易显噱头；分享与私密需平衡。  

### 10.3 玩法与叙事风险（综合）

- 机制「失败感」需与**庄重收束**（葬礼、冥想、终局）配套，避免单纯挫败。  
- 手机与工作消息设计需真实但不**羞辱**用户的职业与生活；**花钱选项**需避免引导违法或伤害他人的行为，并保持黑色幽默边界。  

### 10.4 情绪安全强化（V2）

- 叙事避免自杀、自伤、暴力侵害等直接呈现（与「黑色幽默但不恶意」一致）。  
- **自杀意念关键词**：中止流程，进入安静求助页，提供如 **400-161-9995** 等热线资源，不可轻率跳过关闭逻辑由产品定义。  
- **语气偏好**：犀利 / 温和两档，影响死神旁白与叙事措辞的攻击性（仍不羞辱用户生活与职业）。

## 11. AI 的角色定位（综合定稿）

### 11.1 原则

- **预制骨架保体验下限**；AI 做**精准插入**，而非通篇依赖长生成。  
- 玩家看到的大量为打磨过的模板句，AI 在批注、死因选择、**场所旁白与三选项**、**自定义行动后的续写**、**消息与朋友圈文案**、**死神花钱后果**、悼词、冥想回扣等处注入个性化。  

### 11.2 职责对照（摘要）

| 环节 | AI 侧重 | 预制侧重 |
|------|---------|----------|
| 入殓登记 | 死神批注 | 表格 UI、问题文案 |
| 死因 | 从库中选隐喻匹配项 | 死因库、标签 |
| 遗愿 | 分类理解、必要时 3 条建议 | 引导文案 |
| 最后一天场所 | 旁白、随机耗时区间内的具体分钟数、三选项文案；自定义行动的续写与时间估算 | 绿色通道 30 分规则、随机耗时标签表、骨架模板 |
| 手机 | 收发消息文案、回复延迟由随机数驱动后的生成；朋友圈内容 | UI、**100% 起算**、**亮屏 3 游戏分/1%**、**充电宝充电 −1 游戏小时与回电数值表**、延迟 0～180 游戏分规则 |
| 现金 | 花钱后果叙事、余额校验 | 5000 元初始值、死神花钱菜单与扣款逻辑 |
| 葬礼 | 悼词、墓志铭候选 | 流程、视觉模板 |
| 冥想 | 关键意象回扣 | 主体结构与时长 |

### 11.3 回声机制（V2）

用户原话建议在体验中**至少三次**回响，例如：登记表「改天再说」、葬礼悼词、冥想尾声引用核心输入——逐步加深「这是我说过的」的触感。

## 13. 术语说明

- **灵魂画像（Soul Profile）**：入殓登记三问输出，用于驱动场所、消息隐喻与悼词引用。  
- **遗愿**：用户在最后一天唯一核心想完成的那一件事（表达上仍称「遗愿」）。  
- **游戏内时间**：与真实时间不同步的加速时间；**恒逝**，不因界面停顿而停止（菜单策略除外）。  
- **绿色通道**：死神提供的传送，**固定消耗 30 分钟游戏时间**抵达所选场所。  
- **最后一笔现金**：本局初始 **5000 元**虚构现金，通过死神花钱选项消耗。  
- **手机回复延迟**：用户发消息后，在请求回复生成时随机 **0～180 分钟（游戏时间）** 后收到回复。  
- **亮屏耗电**：亮屏状态下 **每 3 分钟游戏时间扣 1% 电量**（自 100% 起算）。  
- **充电宝充电**：用户主动选择，**消耗 1 小时游戏时间**为手机补电，具体恢复量见数值配置。  
- 其余术语同原设计：**时间块**（可用游戏内时段替代表述）、**葬礼通知/讣告卡**、**死亡冥想**；终局阶段的个人书写可称**体验简报**（非多局累积日记）。
