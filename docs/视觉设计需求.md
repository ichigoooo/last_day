# 《人生最后一天》视觉设计需求文档

> 用途：供美术与生成图工作流对齐界面结构与素材缺口。  
> 画布：竖屏 **1080×1920**（`canvas_items` 拉伸）。  
> 说明：**不包含风格提示词**；风格由你方通过生图 Agent 自行选定。UI 控件（按钮、输入框等）可后续单独抠图或矢量重绘，本文侧重「场景级 / 插画级 / 装饰级」素材需求。

---

## 1. 产品概述

### 1.1 定位与用途

《人生最后一天》（Last Day Simulator）是一款 **Godot 4.6 + C#** 的互动叙事体验：以「生命最后一天」为设定，引导玩家完成一次带仪式感的自我审视。体验为 **单次完整流程**，不强调多局累积。

### 1.2 核心玩法与流程

玩家在流程中会经历：**主菜单与 API 配置** → **入殓登记**（书写自我画像）→ **死因宣判**（死神对话）→ **最后一天**（核心：时间流逝、现金预算、手机电量、自然语言行动与选项叙事）→ **死亡过渡（黑屏）** → **终章归档（葬礼阶段：遗愿、系统报告、讣告、墓志铭、留档卡片）** → **冥想回执** → **回到现实（终局）**。

可选能力：用户自备 **LLM API**，用于个性化叙事与视觉 brief 生成；未配置时使用本地兜底文案与极简图形。**危机关键词**会触发全屏求助层（心理热线），并中断体验返回主菜单。

### 1.3 气质与内容约束（供美术理解语境，非画风指令）

- 叙事语气：冷静、克制、略带荒诞；**清醒但不伤人**。  
- 中文界面与叙事为主。  
- 自杀/自伤相关输入会触发合规求助界面（见「危机求助层」）。

---

## 2. 全局视觉与 UI 关系

### 2.1 程序内主题（协调参考）

当前代码中全局 UI 以 **深色底、冷灰蓝描边、克制金 hover、低饱和红作按压反馈** 为主（见 `scripts/ui/AppTheme.cs`）。若你方提供 **全屏背景图 / 大插画**，需预留 **顶部状态区、中部主视觉区、底部叙事与输入区** 的对比度与可读性；或提供可切九宫格的 panel 边框与暗角素材，便于与现有控件叠加。

### 2.2 与程序生成视觉的关系

「最后一天」主界面中，**场景卡片**与**人物卡片**当前可由 **LLM 生成 SVG** 动态贴图（`VisualSvgService`）；地图选点则另有 **场所 SVG** 管线（`LocationCardSvgService`）。美术清单中仍列出 **可选替换位**：若你希望统一手绘/3D 风格，可用 **静态图集 + 命名规范** 替换程序化贴图（需程序配合改绑定，非本文范围）。

## 3. 分界面构成与素材需求

下列「界面」与 `GameManager.Phase` / 场景文件对应；**组成要素**按当前场景与脚本中的实际控件整理。

---

### 3.1 主菜单（`scenes/main.tscn`）

**流程角色**：入口；检查 API 是否配置；进入登记或设置。

**组成要素**

- 全屏容器；纵向 **VBox**：主标题、说明文案（API 配置提示）、横向按钮行（「API 设置」「开始体验」）、「测试 API」按钮。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 主菜单背景 | 全屏底图或可分层的背景（可带轻微纹理/噪点），需保证标题与按钮区对比度 |
| 标题装饰（可选） | 与标题排版配合的装饰线、印章感图形、极简符号（非必须） |
| Logo / 应用图标 | `project` 已引用 `icon.svg`，若需商店/启动统一视觉可提供独立画稿 |

---

### 3.2 API 设置（`scenes/settings.tscn`）

**流程角色**：填写 API Key、Base URL、模型名；保存后标记首次设置完成。

**组成要素**

- `PanelContainer` 内表单：多组标签 + `LineEdit`、状态标签、保存按钮。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 设置页背景 | 可与主菜单同系列，或略「更工具感」的底图 |
| 表单面板装饰（可选） | 面板衬底、细边框、角标（与 AppTheme 面板色可协调） |

---

### 3.3 入殓登记（`scenes/death_registration.tscn` + `DeathRegistrationScreen.cs`）

**流程角色**：三问（工作/关系/逃避）→ 提交 → 灵魂标签与死因等由系统处理 → **Dialogic 时间线**呈现批注与过渡 → 进入宣判。

**组成要素**

- 场景内曾含传统表单块与打字机批注区；当前流程以 **深色全屏底 + Dialogic 演出** 为主（部分 legacy 控件在脚本中隐藏）。
- 需考虑 **Dialogic 对话层** 的视觉占位：立绘、文本框、姓名条（若时间线中使用）。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 登记阶段背景 | 偏「档案/登记处/冥界前台」意象的底图（抽象即可），用于 Dialogic 全屏或半屏 |
| 死神/办事员立绘（可选） | 若 Dialogic 时间线使用角色立绘，需正面或半身、透明底 PNG/SVG 序列 |
| 对话 UI 框体（可选） | 文本框、选项条、立绘槽位装饰，与 Dialogic 样式配置对齐 |
| 档案纸张纹理（可选） | 用于批注或表单回忆闪回时的叠层 |

---

### 3.4 宣判（`scenes/verdict.tscn` + `VerdictScreen.cs`）

**流程角色**：展示死因与死神开场白 → **Dialogic 时间线**结束后进入「最后一天」。

**组成要素**

- 纸质风格 `PaperPanel` 在代码中可被隐藏，实际以 **Dialogic** 为主；变量注入：`death_cause_text`、`reaper_opening`。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 宣判场景背景 | 与登记区分镜：更「裁定/法庭/渡口」意象；或同一世界观的另一角度 |
| 死神立绘差分（可选） | 宣判语气用的表情或姿态差分 |
| 裁定书/令纸视觉元素（可选） | 条漫式竖条、火漆、印章图形（作局部贴图即可） |

---

### 3.5 最后一天（`scenes/last_day.tscn` + `LastDayScreen.cs`）——内容最多

**流程角色**：核心玩法；状态栏显示游戏内时间、阶段、现金、电量；主区展示 **场景插画 + 人物插画**（现为动态 SVG 位）、叙事文本、选项与自由输入；工具栏打开 **地图**、**消费**、**手机** 等浮层；临近结束时可有 **濒死遮罩**。

**组成要素（主界面）**

- 顶部 **StatusBar**：时间标签、相位标签、现金、电量条；左侧可有 **ClockArc** 占位控件（可做环形时间意象）。
- **Toolbar**：地图按钮、结束对话按钮、当前地点标题。
- **MainVisual**：横向 **SceneCard**（场景图）+ **PortraitCard**（人物图）+ **CharacterCaption**。
- **SummaryLabel**、**NarrationScroll**（富文本叙事）、**OptionsColumn**（三选项按钮）、自定义行动提示、**InputRow**（头像占位 `AvatarStub`、输入框、发送）。
- **ToolRow**：打开手机、打开消费。
- **NavRow**：返回 / 下一步（流程用）。
- **PhoneLayer**：手机面板——聊天、发送、充电宝、朋友圈输入与列表等。
- **SpendLayer**：消费列表与结果展示。
- **MapLayer**：地图标题、地点 **Grid** 选点。
- **DyingFxLayer**：全屏 **ColorRect** 濒死效果（可改为带纹理或渐变的叠层图）。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 主界面背景 | 全天底图或分时段底图（清晨/午后/傍晚/深夜），与 `TimeManager` 相位可联动（若程序支持换图） |
| 场景卡片图（8 类场所） | 与 `LocationManager` canonical id 对齐：**home / office / park / hospital / bar / school / seaside / cemetery**；每类至少一图，供替换 `SceneCard` |
| 人物/遭遇立绘占位 | 通用「路人」「亲友」「死神」等剪影或风格化人像，供替换 `PortraitCard`；可按角色 id 扩展 |
| 地图插画 | 一张「城市/抽象路网」底图 + 8 个地点 **图标或图钉**（选中/未选中两种状态更佳） |
| 手机外壳与内屏 | 手机浮层可叠加：外框、刘海区、聊天气泡装饰（内容仍为控件） |
| 消费面板装饰 | 小票、账单、支付成功/失败意象的顶条或角标 |
| 电量与现金图标 | 电池图标、硬币/纸币小图标（状态栏内嵌，可与现有 ProgressBar 组合） |
| 时钟/日相图标 | 供 ClockArc 或时间旁展示（太阳/月亮/刻度） |
| 濒死遮罩 | 暗角、血雾、噪点、Vignette 纹理（需中间留叙事可读区） |
| 玩家头像 | 替换 `AvatarStub` 色块（圆形或圆角方） |

---

### 3.6 死亡过渡（`scenes/death.tscn` + `DeathScreen.cs`）

**流程角色**：**24:00 后纯黑静默**若干秒，自动进入葬礼场景。

**组成要素**

- 全屏黑色 `ColorRect`，无交互叙事。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 纯黑过渡（可选） | 若需「非纯黑」：极暗噪点、呼吸感微光（需极弱，不破坏静默感） |
| 音效配合（非美术） | 文档外：可单独列音效需求 |

---

### 3.7 终章归档 / 葬礼（`scenes/funeral.tscn` + `FuneralScreen.cs`）

**流程角色**：滚动长页：**遗愿登记** → **最终系统报告** → **讣告（打字机）** → **墓志铭校对** → **留档卡片（ShareCard）** → 进入冥想。

**组成要素**

- 深色背景 + 全屏 **ScrollContainer**；多段 **章节标题**、正文、按钮；**ShareCard** 为带圆角边框的深色「讣告卡」面板（程序内 `StyleBoxFlat` 绘制）。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 葬礼/归档背景 | 低对比纹理或渐变，长页滚动时上下可衔接 |
| 章节分隔装饰 | 细线、符号、小图标（遗愿/报告/讣告/刻字/卡片各一段） |
| 讣告卡边框纹理 | 可替换 `ShareCard` 纯矢量边框的纸质/金属压印质感（需留足内边距给动态文本） |
| 打字机区域装饰（可选） | 轻量电影胶片、扫描线（勿挡字） |

---

### 3.8 冥想 / 回执（`scenes/meditation.tscn` + `MeditationScreen.cs`）

**流程角色**：展示「真相揭示」长文（AI 或兜底）→ 追问「明天第一件想做得不一样的事」→ 可选留白 → 进入终局。

**组成要素**

- 深色背景 + 标题「回执」+ **ScrollContainer** 正文 + 输入行 + 「留白」「回到现实」按钮。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 冥想背景 | 与葬礼区分：更「空、静、黎明前」的抽象场景或单色晕染 |
| 回执纸意象（可选） | 轻纹理衬底，突出「官方回执」感但不过度喜庆 |

---

### 3.9 回到现实 / 终局（`scenes/ending.tscn` + `EndingScreen.cs`）

**流程角色**：浅色背景；展示收束文案与玩家留言（若有）；静默写入本地 txt；**返回主菜单**。

**组成要素**

- 浅米色全屏底 + 大标题「回到现实」+ 多行说明 + 单按钮。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 终局背景 | **浅色**、暖灰或旧纸感，与全局深色界面形成情绪收束 |
| 晨光/窗光（可选） | 顶部或侧向柔光纹理，象征返回日常 |

---

### 3.10 危机求助层（`scenes/crisis_help_overlay.tscn`）

**流程角色**：检测到危机内容时 **全屏遮罩**，展示热线与说明，暂停游戏树；**返回主菜单**。

**组成要素**

- 半透明深色 **Dim** + 居中 **VBox**：标题「你值得被倾听」、正文（热线 **400-161-9995**）、按钮。

**建议美术素材**

| 类型 | 说明 |
|------|------|
| 求助页背景 | 严肃、安全、无惊悚元素；可与主菜单完全不同的「援助」色系 |
| 小图标（可选） | 电话、倾听、护盾等低刺激符号 |

## 5. 交付与规格建议（非风格）

- **分辨率**：主背景建议 **1080×1920** 或更高一倍供缩放；卡片类插图可按 **约 512～1024 像素短边** 试作，以不糊为准。  
- **格式**：背景可用 **PNG / WebP**；需透明叠加的用 **PNG**。矢量 UI 装饰可用 **SVG**。  
- **安全区**：竖屏底部预留 **约 120～180 px** 给系统手势与拇指区；重要信息避开顶栏刘海区（若 targeting 真机）。  
- **命名（建议）**：`bg_main_menu`、`bg_last_day_dawn`、`loc_home`、`loc_office`、…、`portrait_generic_01`、`map_pin_active` 等，便于程序对接。
