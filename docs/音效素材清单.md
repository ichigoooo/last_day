# 音效素材清单（全面增强体验）

本文档用于规划 **短音效（SFX）** 素材：用途说明、与工程一致的**文件命名**（即 `AudioManager.PlaySfx` 的 key），以及**时长建议**。  
当前工程约定：文件放在 `res://assets/audio/sfx/`，文件名为 **`<key>.wav`**（小写蛇形命名）。

---

## 总则

| 项目 | 说明 |
|------|------|
| 格式 | 优先 **WAV**（与 `AudioManager` 加载路径一致）；立体声或单声道均可，手机竖屏建议单声道也可接受 |
| 命名 | **蛇形小写** `snake_case`，与下表「建议文件名（key）」列一致，扩展名 `.wav` |
| 电平 | 统一做轻度归一化，避免个别素材过响；预留与 BGM 混音的余量 |
| 时长 | 表中为 **建议区间**；循环类需首尾可无缝或淡入淡出友好 |

**时长分级（便于采购与剪辑）：**

- **极短**：约 0.05～0.25s — 点击、微反馈  
- **短**：约 0.25～0.8s — 常规 UI、单次动作  
- **中**：约 0.8～3s — 盖章、提示音、转场点缀  
- **长/可循环**：约 4～60s — 氛围铺底（需标注是否 loop）

## 七、实施与扩展说明

1. **与代码对接**：在 `AudioManager` 中为每个 key 增加 `public const string SfxXxx = "key";`，界面处调用 `PlaySfx(AudioManager.SfxXxx)`。  
2. **同一动作多条随机**：如脚步声，可用 `footstep_indoor_01`、`footstep_indoor_02`，代码侧随机选 key。  
3. **氛围类**：若文件较长，可在 Godot 中把 `AudioStream` 设为循环，或拆成「可拼接」的短段。  
4. **优先级**：先补齐 **第一节已引用** 的 4 个文件，再按阶段优先补 **最后一天** 与 **全局 UI**，最后补环境与叙事增强类。
