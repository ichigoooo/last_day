# 手动验收清单（6.2–6.4）

以下需在 Godot 编辑器中实际跑通；完成后将 `tasks.md` 中 6.2–6.4 勾选为 `[x]`。

## 6.2 三项核心目标

- [ ] **开头不再像表单**：从主菜单进入序章，确认以 Dialogic 对白与 `text_input` 采集为主，无三块大表单主导流程。
- [ ] **终章不再像 wizard**：从最后一天玩到 `Ending`，确认葬礼 / 冥想 / 结束为连续仪式，无「下一步」堆砌感。
- [ ] **中段自由度未受损**：在 `LastDay` 确认手机、地图、现金、对话等仍可正常使用；`archive_summary` 类锚点生效且无强制遗愿字段。

## 6.3 异常路径

- [ ] **无 API / 未配置 Key**：清空或无效 `user://settings.json` 中的 API 配置，走完全程，确认降级文案完整、语气成立。
- [ ] **低质量 / 空返回**：可在断网或错误 Base URL 下抽样验证序章批注、死因、开场是否有 Fallback。
- [ ] **中断切场**：序章或宣判 Dialogic 进行中返回主菜单再进入；终章中途退出再续（若项目支持）— 按设计预期无卡死、无重复信号。

## 6.4 终章情绪与媒介

- [ ] **按钮**：每个可点控件是否只在需要时出现，无功能堆砌。
- [ ] **停顿与留白**：黑屏、静音、长停顿是否服务于情绪。
- [ ] **音效**：是否衬托节点而非提示「你点了按钮」。
